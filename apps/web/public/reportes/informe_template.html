Detectado: faltanteNO CONFORME

RIESGOSin CSP se pierde control sobre los recursos que ejecuta el navegador, incrementando el riesgo de ejecución de scripts no autorizados (XSS) y fuga de información.

Cabeceras de endurecimiento

OBSERVACIONES · CORTO PLAZO

X-Frame-Options

OBSERVACIÓN
Añadir SAMEORIGIN (o DENY) para impedir que el sitio se embeba en iframes de terceros.
Nginx add_header X-Frame-Options "SAMEORIGIN" always;
RIESGOIncrustación del sitio en dominios externos: suplantación de interfaz (clickjacking).

X-Content-Type-Options

OBSERVACIÓN
Añadir nosniff para que el navegador no adivine tipos MIME.
Nginx add_header X-Content-Type-Options "nosniff" always;
RIESGOInterpretación de archivos con tipo distinto al declarado: ejecución de contenido malicioso (MIME sniffing).

Referrer-Policy

OBSERVACIÓN
Añadir strict-origin-when-cross-origin para no filtrar URLs internas a terceros.
Nginx add_header Referrer-Policy "strict-origin-when-cross-origin" always;
RIESGODivulgación de direcciones internas e identificadores de sesión hacia destinos externos.

Permissions-Policy

OBSERVACIÓN
Desactivar APIs sensibles (cámara, micrófono, geolocalización) que no se usan.
Nginx add_header Permissions-Policy "camera=(), microphone=(), geolocation=()" always;
RIESGOComponentes de terceros pueden acceder a dispositivos sensibles, debilitando la privacidad del usuario.

Cross-Origin-Opener-Policy

OBSERVACIÓN
Añadir same-origin para aislar la ventana de otros orígenes.
Nginx add_header Cross-Origin-Opener-Policy "same-origin" always;
RIESGOContexto de navegación accesible a otras ventanas: exposición a ataques de canal lateral (Spectre).

Cross-Origin-Resource-Policy

OBSERVACIÓN
Añadir same-origin para controlar quién puede cargar los recursos del sitio.
Nginx add_header Cross-Origin-Resource-Policy "same-origin" always;
RIESGORecursos del dominio solicitables desde orígenes externos no autorizados.
